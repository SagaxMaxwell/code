# ORM 笔记

#### 什么是持久化

- 定义，把数据永久保存在存储设备中
- 应用，将内存中的数据存储在关系型的数据库中、磁盘文件中、XML 等文件中

#### 什么是持久层

- 实现持久化应用领域的某个特定系统的逻辑层面，将数据使用者和数据实体相关联

#### 持久化层的不足

- 持久化层缺乏弹性，业务出现变更就必须修改持久化层的接口
- 持久化层绑定域模型、关系数据库模型，不论哪者发生变化都要修改持久化层的代码，增加了维护难度

#### 什么是 ORM

- 定义，解决面向对象和关系数据库互不匹配的技术
- 通俗说，是通过描述对象和数据库之间映射的元数据，将程序中的对象自动持久化到关系数据库中
- 使用时，不需要编写复杂的 SQL 语句，就与平常操作对象一样

#### ORM 方法论基于什么原则

- 简单性，以最基本的形式建模数据
- 精确性，基于数据模型创建了标准化的结构
- 传达性，数据库结构被任何人都能理解的语言文档化

#### 为什么要做持久化和 ORM 设计

- MVC 模式将表现层（即 View）和数据处理层（即 Model）的解耦合，而持久化的设计则实现了数据处理层内部的业务逻辑和数据逻辑分离的解耦合

#### ORM 设计的优缺点

- 优点
	- 开发效率更高，不用再关注数据库细节，不需要写 SQL 语句
	- 数据迁移方便，修改数据库只需要该配置，不需要管数据库之间的语法差异
	- 支持面向对象封装
- 缺点
	- 性能低，内部采用了大量反射和数据库检测，降低程序的执行效率
	- ORM 常为多层结构，层次多了效率会低下
	- ORM 不能生成所有的业务语句，如复杂的报表生成不了，需要编写 SQL 语句

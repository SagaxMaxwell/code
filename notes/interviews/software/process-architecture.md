# 软件过程与架构面试笔记

#### 开发模型

| 开发模型 | 特点简述                           |
| ---- | ------------------------------ |
| 瀑布模型 | 阶段分明、顺序执行，适用于明确需求，风险暴露晚        |
| 原型模型 | 快速构建原型，适用于不明确需求，减少变更风险，但迭代成本高  |
| 螺旋模型 | 结合瀑布和原型优点，强调风险分析，适用于高风险项目，管理复杂 |
| 增量模型 | 分阶段实现功能，适用于频繁变更需求，但集成难度大       |
| 迭代模型 | 多周期迭代，适用于需求变更，快速响应，提高成功率       |
| 敏捷开发 | 团队协作、快速迭代，适用于频繁变更和短期项目，注重客户满意度 |

#### 架构设计

- MVC
	- 模型层 (Model)
		- 应用程序的核心数据和业务逻辑，可被多个视图和控制器使用
	- 视图层 (View)
		- 视图层负责显示数据，不包含任何业务逻辑，从模型获取数据并展示
	- 控制器 (Controller)
		- 负责控制用户的输入，并调用相应的模型和视图，是模型和视图的桥梁
- MVP
	- 与 MVC 类似，但将控制器替换为呈现器，这使得视图和模型之间的耦合度更低
- RESTful
	- 一种网络架构风格，用于设计网络服务，它强调使用无状态的通信协议，并采用统一的资源表示

#### 软件开发周期

- 任何模型都有以下软件开发周期
- 需求获取/分析（系统分析、软件分析）
- 设计
- 实现
- 测试
- 发布
- 维护

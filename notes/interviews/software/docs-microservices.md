# 项目文档与微服务笔记

#### 项目文档

| 文档名称    | 作用                                             | 编写时机                    |
| ------- | ---------------------------------------------- | ----------------------- |
| 需求分析文档  | 明确项目需求，包括功能需求、性能需求、用户需求等，为项目提供方向和目标            | 在项目启动阶段，初步了解项目背景后立即开始编写 |
| 系统详细文档  | 详细描述整个系统的功能、架构、模块划分、性能要求、安全性要求等，为项目开发提供宏观指导    | 项目需求分析完成后，进入系统设计阶段前     |
| 模块设计文档  | 详细描述每个模块的功能、接口、内部数据结构、算法和流程，为开发阶段提供重要参考        | 系统详细文档完成后，针对每个模块进行设计时   |
| 数据库设计文档 | 详细描述数据库的结构、表设计、字段定义、索引、触发器、存储过程等，为数据库开发和维护提供依据 | 模块设计文档完成后，进行数据库设计时      |
| 接口设计文档  | 详细描述不同模块之间的接口定义，包括输入输出参数、数据格式、调用方式等，指导模块间协作    | 模块设计文档完成后，进行接口设计时       |
| 甘特图     | 展示项目中每个任务的开始和结束时间，以及它们之间的关系，帮助项目团队了解项目进度和时间安排  | 项目计划阶段，需求分析完成后，制定项目计划时  |
| 测试计划文档  | 详细描述测试策略、测试方法、测试用例、测试环境和测试工具等，为项目测试提供指导        | 在项目开发阶段，开发工作启动前         |
| 用户手册    | 提供系统操作指南，帮助用户了解如何使用系统                          | 在项目开发完成后，系统准备上线前        |
| 维护手册    | 提供系统维护指南，包括系统配置、故障处理、升级流程等                     | 在项目开发完成后，系统上线后          |
| 项目总结报告  | 总结项目过程中的经验教训，为未来项目提供参考                         | 项目结束后，进行项目总结时           |

#### Python + Nacos 微服务应用

- Nacos（Naming and Configuration Service）是一个更易于构建云原生应用的动态服务发现、配置管理和服务管理平台

| 步骤    | 描述                              |
| ----- | ------------------------------- |
| 环境准备  | 安装 Python 和 Nacos 客户端库          |
| 服务注册  | 注册服务，包括服务名称、IP 地址、端口等           |
| 服务发现  | 发现服务，根据服务名称或标签获取服务实例列表          |
| 配置管理  | 从 Nacos 获取和更新配置                 |
| 服务调用  | 通过服务名或 IP 地址进行服务调用，Nacos 处理负载均衡 |
| 监控与治理 | 使用 Nacos 监控服务状态，管理服务版本          |
| 部署    | 部署 Python 微服务到 Nacos 支持的运行环境    |
| 测试    | 在本地和生产环境中测试微服务                  |
| 运维    | 使用 Nacos 监控和日志功能，滚动升级服务         |

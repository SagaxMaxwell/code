# 数据结构与 C/C++ 基础面试笔记

#### 堆和栈

| 特性   | 栈 (Stack)       | 堆 (Heap)         |
| ---- | --------------- | ---------------- |
| 分配速度 | 快               | 较慢               |
| 访问速度 | 通常比堆的访问速度快      | 访问速度相对较慢         |
| 大小限制 | 大小有限，通常在几 MB 左右 | 大小相对较大，受系统虚拟内存限制 |
| 灵活性  | 较低，固定分配和释放顺序    | 较高，可以动态分配和释放     |
| 碎片化  | 不会产生内存碎片        | 容易产生内存碎片         |

#### 堆的内存分配策略

- 首次适配，从空闲列表的开始位置查找，直到找到第一个足够大的空闲块满足内存请求
- 最佳适配，在空闲列表中查找刚好满足请求的最小内存块
- 最坏适配，在空闲列表中查找满足请求的最大的内存块

#### 红黑树

- 规则
	- 节点是红色或黑色
	- 根节点与叶节点是黑色
	- 红色节点的子节点都是黑色
	- 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点
- 用途
	- 主要用于数据库索引和内存管理中，特别是在多线程环境下
- 优点
	- 平衡性，确保了树的搜索、插入和删除操作的时间复杂度为 `O(log n)`，即使树变得不平衡
	- 多线程安全
- 缺点
	- 空间复杂度，红黑树需要额外的空间来存储颜色信息
	- 规则复杂，操作复杂
	- 路数比较小导致深度越深，查询速度越慢

> 红黑树属于平衡二叉树

#### 宏定义与内联函数

| 特性       | 宏定义 (Macros)                               | 内联函数 (Inline Functions)               |
| -------- | ------------------------------------------ | ------------------------------------- |
| 定义方式     | ` #define MACRO_NAME (args) replacement  ` | `inline type function (args) { ... }` |
| 展开时机     | 编译前，由预处理器展开                                | 编译时，由编译器决定是否展开                        |
| 类型和参数安全性 | 无类型和参数检查，可能导致类型错误                          | 进行类型和参数检查，更安全                         |
| 性能       | 通常没有函数调用的开销                                | 减少了函数调用的开销，但可能不如宏定义快                  |
| 代码膨胀     | 可能                                         | 可能                                    |
| 可读性和维护性  | 低，特别是复杂的宏                                  | 高，是真正的函数                              |
| 使用场景     | 简单的文本替换或常量定义                               | 需要类型检查和安全性考虑的复杂操作                     |

#### 引用和指针

| 特性/概念 | 指针（Pointer）              | 引用（Reference）     |
| ----- | ------------------------ | ----------------- |
| 定义    | 存储变量地址的变量                | 另一个变量的别名          |
| 语法    | `int* ptr;`              | `int& ref = var;` |
| 初始化   | 可以在声明后赋值                 | 必须在声明时初始化         |
| 重新赋值  | 可以重新指向另一个变量              | 一旦初始化后不能更改        |
| 空值    | 可以是空（`nullptr` 或 `NULL`） | 不能为空              |
| 解引用   | 需要使用 `*ptr` 来访问值         | 直接使用，不需要特殊操作符     |
| 指针算术  | 可以进行加、减等操作               | 不支持算术操作           |
| 内存管理  | 用于动态内存分配和释放              | 不用于内存管理           |
| 使用场景  | 需要操作内存地址或需要动态改变指向        | 需要简化代码并固定引用对象     |

# Linux 面试笔记

### Linux 概述

#### 什么是 Linux

- 开源的操作系统
- 支持多用户、多任务、多线程、多 CPU
- 通常 Web 项目部署在 Linux 上

#### Windows 和 Linux

- Linux 适合开发和部署，Windows 适合娱乐办公
- Linux 开源，Windows 不开源

#### Unix 和 Linux

- Linux 开源，Unix 不开源
- Linux 跨平台比 Unix 要好

#### Linux 内核

- 硬件设备管理
- 系统内存管理
- 文件系统管理
- 应用程序管理

#### Linux 组件

- 内核
- 程序
- Shell
- GUI

#### Linux 体系

- 用户空间
	- 用户应用程序
	- C Library
- 内核空间
	- 系统调用接口
	- 内核

> 用户态可转移为内核态（系统调用、硬件中断、异常）
> 不同模式下 CPU 可以执行的指令和访问的寄存器不同
> Linux 从 CPU 的角度出发，为了保护内核的安全，把系统分成了 `2` 部分

#### BASH 和 DOS

- 大小写
	- BASH 区分大小写，DOS 不区分大小写
- 转义符
	- BASH 使用 `\`，DOS 使用 `/`
- 目录分隔符
	- BASH 使用 `/` ，DOS 使用 `\`

> BASH 和 DOS 都是命令行解释器

#### Linux 开机启动过程

- 固件阶段：BIOS/UEFI 完成硬件初始化并选择启动设备。
- 引导阶段：Bootloader（常见为 GRUB）加载 Linux 内核和 initramfs。
- 内核阶段：内核初始化驱动、内存、文件系统等资源。
- 用户态阶段：启动 PID 为 `1` 的初始化进程，现代发行版多为 `systemd`。
- 服务阶段：`systemd` 按 target 启动系统服务，最后进入登录界面或图形界面。

#### Linux 系统缺省的运行级别

- `0`，关机
- `1`，单机用户模式（常用于系统维护）
- `2`，字符界面的多用户模式（不支持网络）
- `3`，字符界面的多用户模式（带网络功能）
- `4`，预留或自定义使用
- `5`，图形界面的多用户模式
- `6`，重启

> 系统缺省，指的是系统或软件在未收到用户明确指示的情况下，自动采用默认设置或行为
> 可在没有用户自定义的情况下，确保系统或软件能够正常运行

#### Linux 进程间的通信方式

- 共享内存
- 消息队列
- 套接字
- 管道
- 信号

#### Linux 重要的系统日志文件

- `/var/log/messages`

> 是许多日志的汇总，内容可看出任何入侵企图或成功入侵的信息

#### 交换空间

- 用于临时保存某些并发运行的程序
- 当内存不足以容纳正在执行的所有程序时，就会发生这种情况

#### Root 账户

- 系统管理员账户，允许完全控制系统

#### CLI (CUI) 和 GUI

- CLI
	- 命令行界面
	- 不支持鼠标，通过键盘输入指令
- GUI
	- 图形用户界面
	- 支持鼠标和键盘，操纵图标或菜单选项

> CLI 也称作字符用户界面 (CUI)

#### 开源的优势

- 公布源代码
- 社区可添加功能、更正源代码

#### GNU

- 自由软件项目
- 目标是创建完全由自由软件组成的操作系统

### 文件系统

#### Linux 中的文件

- 所有被操作系统管理的资源（包括目录）都视为文件
- 支持 `5` 种文件类型

| 文件类型          | 描述          | 示例              |
| ------------- | ----------- | --------------- |
| 普通文件          | 辅助存储设备存储数据  | 文本文件、二进制文件      |
| 目录文件          | 管理系统中的文件的文件 | `/root`、`/home` |
| 链接文件          | 用于不同文件下的共享  | 硬链接、软链接         |
| 设备文件          | 访问硬件设备      | 键盘、打印机          |
| 命名管道 (`FIFO`) | 进程通信通过该文件完成 |                 |

> 进程通信中的 `FIFO` 并非启动过程中的 `FIFO`

#### Linux 目录结构

| 目录      | 作用                                    |
| ------- | ------------------------------------- |
| `/bin`  | 存放二进制可执行文件，有常用的命令，如 `ls`              |
| `/etc`  | 存放系统管理和配置文件                           |
| `/home` | 存放所有用户文件的根目录                          |
| `/usr`  | 存放系统应用程序                              |
| `/opt`  | 存放额外安装的应用程序                           |
| `/proc` | 虚拟文件系统目录，是系统内存的映射                     |
| `/root` | 超级用户（系统管理员）的主目录                       |
| `/sbin` | 存放二进制可执行文件，但 root 用户才可访问，如 `ifconfig` |
| `/dev`  | 存放设备文件                                |
| `/mnt`  | 超级用户安装临时文件系统的安装点，让用户临时挂载其他的文件系统       |
| `/boot` | 存放系统引导时的各种文件                          |
| `/lib`  | 存放系统运行的相关库文件                          |
| `/tmp`  | 存放各种临时文件，公用临时文件存储点                    |
| `/var`  | 存放运行时需要改变的文件，也是某些大文件的溢出区，如系统日志        |

#### 索引节点 (inode)

- 文件存储硬盘上，硬盘最小存储单位称为扇区 (`0.5 KB`)
- 由多个扇区可视为块 (block)，是文件系统管理数据的基本单位
- 文件数据存储在块中，但文件的元数据存储在索引节点 (inode)
- 每个文件都有对应的 inode

> 索引节点使用引用计数机制，当为 `0` 时，才会真删除

#### 硬链接和软链接

- 硬链接 (Hard Link)
	- 指向 inode 的指针
	- 不能在不同文件系统的文件间建立硬链接
	- 硬链接不能指向目录
- 软链接 (Symbolic Link)
	- 创建新的 inode
	- 是特殊的文件，数据指向目标文件的路径
	- 当原文件不存在或路径发生变化时，inode 会变成悬挂链接

```bash
# 硬链接
ln target_file hard_link

# 软链接
ln -s target_file symbolic_link
```

#### 文件权限

- `chmod`，更改文件或目录的权限
- `chown`，更改文件或目录的所有者和组
- `chgrp`，仅更改文件或目录的组

```bash
# chmod
# 更改目录或文件的权限
chmod 755 file_name
chmod rwxr-xr-x file_name

# 递归更改文件或目录及其子文件或目录的权限
chmod -R 755 dir_name

# 对文件所有者添加执行权限
chmod u+x file_name


# chown
# 更改文件的所有者和组
chown user_name:group_name file_name

# 仅更改文件的所有者
chown user_name file_name

# 仅更改文件的组
chown :group_name file_name

# 递归更改文件或目录及其子文件或目录的所有者和组
chown -R user_name:group_name dir_name

# 递归更改文件或目录及其子文件或目录的所有者
chown -R user_name dir_name

# 递归更改文件或目录及其子文件或目录的组
chown -R :group_name dir_name


# chgrp
# 更改文件或目录的组
chgrp group_name file_name

# 递归更改文件或目录及其子目录或文件的组
chgrp -R group_name dir_name
```

> 读 (`r`)，`4`
> 写 (`w`)，`2`
> 执行 (`x`)，`1`
> `chmod` 修改的权限顺序是所有者、组内成员、其他用户

### 系统安全

#### Linux 初始化环境后需要做什么安全工作

- 添加普通用户登录，禁止 root 登录
- 配置防火墙
- 将服务器设置为密钥登录，禁止密码登录
- 安装 `fail2ban` 软件，防止 SSH 暴力破解
- Web 层使用 WAF/CDN 做流量过滤，但 SQL 注入的根本防线仍是参数化查询、ORM 绑定参数和最小权限账户
- 设置 Web 服务使用 www 用户启动，更改网站目录的所有者和所属组为 www

#### CC 攻击和 DDOS 攻击

- CC 攻击，模拟多个用户访问网页，消耗系统资源
- DDOS 攻击（分布式拒绝服务攻击），将多个计算机联合起来作为攻击平台，对任意数量目标发动 DDOS 攻击

> 攻击，即是通过大量合法的请求占用大量网络资源，以达到瘫痪网络的目的
> 破解方法，使用硬件防火墙做流量清洗，将攻击流量引入黑洞

#### SQL 注入

- 将恶意的 SQL 代码注入到查询中，从而非法地读取、修改、删除数据库中的数据
- 使用参数化查询、输入校验、最小权限数据库账号和审计日志预防。WAF 只能作为补充，不应替代代码层防护。

#### Shell 脚本

- 是文本文件，以 `.sh` 为后缀，包含任意条命令
- 用于完成日常任务
- 特点
	- 可移植
	- 可重用
	- 自动化

### 实战

#### 选择操作系统版本

- 桌面可选 Ubuntu、Fedora 或本机熟悉的发行版。
- 服务器优先选择有长期维护周期的发行版，例如 Debian、Ubuntu LTS、RHEL 或兼容 RHEL 的发行版。
- 新硬件、新桌面栈或滚动体验可用 Fedora。

#### 如何规划主机

- 确定用途
- 确定合适的系统
- 优化系统的参数

#### 网站慢会如何处理

- 因素
	- 网络带宽低
	- 服务器负载大
	- 数据库瓶颈
	- 代码缺少优化
- 排查方式
	- 看服务器负载情况（网络、CPU、内存）
	- 查看数据库日志、系统日志
- 解决方案
	- 提高网络带宽
	- 数据返回慢，优化查询，使用 Redis 缓存常用数据
	- 优化系统架构
	- 优化代码

#### Linux 性能调优方式

- Disabling daemons (关闭 daemons)
- Shutting down the GUI (关闭 GUI)
- Changing kernel parameters (改变内核参数)
- Kernel parameters (内核参数)
- Tuning the processor subsystem (处理器子系统调优)
- Tuning the memory subsystem (内存子系统调优)
- Tuning the file system (文件系统子系统调优)
- Tuning the network subsystem（网络子系统调优)
